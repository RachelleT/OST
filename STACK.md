# Tech Stack & Setup

## Node version

Use Node 24 LTS (the current Active LTS as of mid-2026, supported through April 2028). Node 22 (Maintenance LTS) also works. Add a `.nvmrc` file at the project root containing `24` so future setups match.

If you don't have Node installed: install the LTS version from [nodejs.org](https://nodejs.org). If you work on multiple Node projects and want to switch versions per project, install **nvm** (Mac/Linux) or **fnm** (Windows) instead.

The reason to pin: when your teammate sets up the project on their machine 3 months from now, they install the same Node version you used and dependencies install identically.

## Isolation model

Node projects are already isolated by default: `npm install` puts everything into `node_modules/` inside the project folder, never on your system globally. There is no "virtualenv" step needed.

Two things to commit to git for reproducibility:
- `package.json` — declares the dependencies
- `package-lock.json` — locks every dependency and sub-dependency to exact versions

Both are created by `npm install`. `node_modules/` is NOT committed (Vite's default `.gitignore` excludes it).

### One-shot install

After running `npm create vite@latest . -- --template react-ts` (which sets up React + TypeScript + Vite + the React-related dev dependencies), install everything else in one go:

```bash
npm install \
  @supabase/supabase-js \
  react-router-dom \
  date-fns \
  framer-motion \
  browser-image-compression

npm install -D \
  tailwindcss \
  postcss \
  autoprefixer \
  vite-plugin-pwa \
  workbox-window
```

`-D` marks dev dependencies (build/tooling only, not needed at runtime).

After install, verify `package.json` has all of these listed and `package-lock.json` exists. If a package fails to install (network error, etc.), stop and report it — don't pick alternates.

### TypeScript types

`@types/react`, `@types/react-dom`, and `@types/node` are pulled in automatically by the Vite template. If for some reason you need to add them manually:

```bash
npm install -D @types/react @types/react-dom @types/node
```

## Environment variables

```
# .env.example
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key

# Used only by edge functions, not the client:
SUPABASE_SERVICE_ROLE_KEY=...
OPENAI_API_KEY=...          # M4
GOOGLE_VISION_API_KEY=...   # M4
VAPID_PUBLIC_KEY=...        # M2
VAPID_PRIVATE_KEY=...       # M2
```

Never reference the service role key from client code. It bypasses RLS.

## Supabase setup

This project uses the **Supabase web dashboard only** — no CLI. All schema changes happen via the SQL Editor. See `SETUP.md` for the step-by-step checklist.

In short:
- The Supabase project already exists
- Migrations from `SCHEMA.md` are run by copy-pasting into the dashboard SQL Editor
- The storage bucket is created via the dashboard Storage UI
- Auth providers configured via the dashboard Authentication UI
- Admin users promoted via the dashboard Table Editor (set `is_admin = true` on the row)

Claude Code does NOT install the Supabase CLI, run `supabase` commands, or attempt to use database connection strings. If a step needs SQL run, Claude Code writes the SQL into a file in `supabase/migrations/` and asks the user to paste it into the dashboard.

## Vercel setup

See `SETUP.md`. In short: connect the GitHub repo to Vercel via the Vercel dashboard, add the two `VITE_*` env vars, deploy. No Vercel CLI needed.

## PWA configuration

Use `vite-plugin-pwa` in `auto-update` mode.

```ts
// vite.config.ts
import { VitePWA } from 'vite-plugin-pwa'

export default defineConfig({
  plugins: [
    react(),
    VitePWA({
      registerType: 'autoUpdate',
      includeAssets: ['icons/*.png'],
      manifest: {
        name: 'One Small Thing',
        short_name: 'One Small Thing',
        description: 'One prompt a day. Build something quiet, just for you.',
        theme_color: '#2DBFA8',
        background_color: '#FFFFFF',
        display: 'standalone',
        orientation: 'portrait',
        icons: [
          { src: '/icons/icon-192.png', sizes: '192x192', type: 'image/png' },
          { src: '/icons/icon-512.png', sizes: '512x512', type: 'image/png' },
          { src: '/icons/icon-512.png', sizes: '512x512', type: 'image/png', purpose: 'maskable' }
        ]
      }
    })
  ]
})
```

Icons: generate from a single 1024x1024 source. Sharp brand mark in the center, single-color (use one of the day palette accents — e.g. teal `#04342C` on light cream `#FAF5EC` background works well for a neutral app icon since it's not Monday-specific).

Daily palette is set via inline CSS variables on the root layout component for the Today screen.

## Things to NOT do

- Don't add Redux, Zustand, or any state library. React state + Supabase realtime is enough.
- Don't add a UI kit (shadcn, Radix, etc.). Build the few primitives we need.
- Don't write your own auth. Use Supabase Auth.
- Don't use Next.js. Vite + React Router is simpler and we don't need SSR.
- Don't write a Capacitor wrapper in v1. PWA install works.
- Don't add analytics in v1. Add Plausible (privacy-friendly, free for small sites) in M5 if needed.
